# 管理后台体验增强

> 状态：已完成

## 背景

管理后台新建文章页面存在三个体验问题：
1. 文章内容只能手动输入，无法导入已有 Markdown 文件
2. 标签区域在数据库无标签时完全空白，也无法新建标签
3. 后台页面仍显示博客前台的顶部导航栏和底部 footer，视觉上与前台混淆

## 方案

### 1. 文件导入

在 PostForm 内容 textarea 上方添加"导入文件"按钮，用隐藏的 `<input type="file" accept=".md,.txt,.markdown">` + FileReader 读取文件内容后填入 textarea。复制粘贴为 textarea 原生能力，无需处理。

涉及文件：`src/components/admin/PostForm.tsx`

### 2. 标签内联新建

在标签 pill 列表末尾添加输入框 + "+" 按钮。输入标签名 → 自动生成 slug → `POST /api/tags` 创建 → 刷新列表并自动勾选。支持 Enter 快捷提交。

已有 API：`POST /api/tags` 接受 `{ name, slug }`，已有 admin 鉴权。

涉及文件：`src/components/admin/PostForm.tsx`

### 3. 管理后台隐藏顶部导航

新建 `LayoutShell` 客户端组件，用 `usePathname()` 判断路径前缀。`/admin` 路径下不渲染 Header / Footer / BackgroundDecor，仅保留 ThemeProvider + body 基础样式。修改根 layout 使用该组件包裹。

另一方案是 route groups 分离 `(blog)` / `(admin)`，但需要移动 6+ 个目录，改动过大，当前选择条件渲染方案。

涉及文件：新建 `src/components/LayoutShell.tsx`，修改 `src/app/layout.tsx`

### 4. 侧边栏添加博客入口

在 Sidebar 导航列表末尾（退出登录按钮之前）添加"查看博客"链接指向 `/`，带外部链接图标。

涉及文件：`src/components/admin/Sidebar.tsx`

## 变更评估

预估文件数：4 个
涉及模块：PostForm、根 layout、Sidebar
影响范围：仅管理后台 UI，不影响前台页面和 API
