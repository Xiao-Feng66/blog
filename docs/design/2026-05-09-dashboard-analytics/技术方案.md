# 仪表盘数据分析

> 状态：待开发

## 背景

管理后台仪表盘目前只展示文章数量（总数、已发布、草稿），缺少访问数据，无法了解博客的实际访问情况和热门内容。

## 需求

在仪表盘展示以下 4 项数据，支持按时间维度筛选（近 7 天 / 近 30 天）：

1. **页面访问量（PV）** — 总页面访问次数
2. **独立访客数（UV）** — 去重后的独立访客数
3. **访问趋势图** — 按天的 PV/UV 折线图
4. **热门文章排行** — 按访问量排序的 Top 10 文章

## 方案

### 1. 数据模型

在 Prisma schema 新增 `PageView` 模型：

```prisma
model PageView {
  id        String   @id @default(uuid()) @db.Uuid
  path      String   @db.VarChar(500)
  postId    String?  @map("post_id") @db.Uuid
  sessionId String   @map("session_id") @db.VarChar(64)
  createdAt DateTime @default(now()) @map("created_at")

  @@index([createdAt])
  @@index([postId])
  @@map("page_views")
}
```

字段说明：
- `path`：访问路径（如 `/posts/my-article`）
- `postId`：关联文章 ID（仅文章详情页有值）
- `sessionId`：访客标识，用 hash(IP + User-Agent + 日期) 生成，每日轮换，不存原始 IP
- `createdAt`：访问时间

### 2. 数据采集

新增 `POST /api/track` 路由：

- 接收 `{ path, postId? }`
- 从请求 headers 提取 IP 和 User-Agent，hash 生成 sessionId
- 写入 `page_views` 表
- 无需鉴权（公开页面都需要上报）
- 返回 204 No Content

在 `LayoutShell` 中添加 `<TrackPageView />` 客户端组件，非 admin 页面加载时自动调用 `/api/track`。文章详情页额外传入 postId。

### 3. 数据查询

新增 `GET /api/analytics?range=7d|30d` 路由（需 admin 鉴权）：

返回结构：
```json
{
  "pv": 1234,
  "uv": 567,
  "trend": [
    { "date": "2026-05-01", "pv": 100, "uv": 50 },
    { "date": "2026-05-02", "pv": 120, "uv": 60 }
  ],
  "topPosts": [
    { "postId": "...", "title": "...", "slug": "...", "views": 200 }
  ]
}
```

趋势数据用 Prisma `$queryRaw` 执行 SQL 聚合（按天 GROUP BY），避免全量加载到内存。

### 4. 仪表盘 UI

保留现有的文章统计卡片，下方新增分析区域：

- **时间筛选器**：近 7 天 / 近 30 天 按钮组
- **PV/UV 统计卡片**：与现有卡片同样式
- **趋势折线图**：使用 Recharts 库（轻量 React 图表库）
- **热门文章排行**：表格，显示排名、标题、访问量

分析区域为客户端组件 `<AnalyticsSection />`，通过 fetch 调用 `/api/analytics` 获取数据，时间切换无需刷新页面。

### 5. Mock 模式

`useMock` 时 `<TrackPageView />` 不上报，`/api/analytics` 返回模拟数据，仪表盘正常展示。

## 变更评估

预估文件数：约 8 个
涉及模块：Prisma schema、API 路由、admin 仪表盘、LayoutShell
影响范围：新增访问追踪（对前台页面加载增加一个异步请求），仪表盘 UI 扩展
新增依赖：recharts
