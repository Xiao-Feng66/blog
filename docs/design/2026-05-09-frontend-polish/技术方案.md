# 前台体验优化

> 状态：待开发 | 最后更新：2026-05-09 10:00:00

## 背景

博客前台存在三个问题：
1. 搜索框过小，搜索结果信息不够丰富，不支持全文搜索
2. 页面背景纯色无层次，视觉单调
3. 夜间模式切换仅影响滚动条，是 Tailwind v4 与 next-themes 配置不匹配的 bug

## 方案

### 一、夜间模式修复

**根因**：Tailwind v4 默认用 `@media (prefers-color-scheme: dark)` 驱动 `dark:` 变体，而 `next-themes` 配置了 `attribute="class"` 在 `<html>` 上切换 `.dark` 类。两者不匹配导致 Tailwind 的 `dark:` 工具类不响应手动切换。

**修复**：在 `globals.css` 中添加：

```css
@custom-variant dark (&:where(.dark, .dark *));
```

涉及文件：`src/app/globals.css`（1 行）

### 二、搜索框重构

#### 2.1 UI 升级

- **尺寸**：搜索面板从 `max-w-md` 加宽到 `max-w-2xl`，输入框高度增大
- **毛玻璃效果**（iOS 风格）：
  - 背景层：`fixed inset-0` 黑色半透明遮罩（`bg-black/40`），加深背景突出搜索框
  - 搜索面板：多层毛玻璃叠加 —— `backdrop-blur-2xl` + 半透明白色/深色背景 + 微妙的边框高光 + 投影
  - 亮色模式：`bg-white/70` 半透明白 + `border-white/50` 高光边框 + `shadow-2xl`
  - 暗色模式：`bg-stone-900/70` 半透明深色 + `border-white/10` 微光边框

#### 2.2 搜索能力增强

- **全文搜索**：API 返回文章 content 字段，FlexSearch 索引包含 `title + summary + content`
- **关键词高亮**：搜索结果中匹配的关键词用 accent 色标记
- **内容片段**：搜索到的结果展示匹配位置附近的文本片段（约 80 字），而非固定摘要

#### 2.3 搜索结果展示

每条结果包含：
- **标题**（高亮匹配词）
- **内容片段**（匹配位置前后的上下文文本，高亮匹配词）
- **标签列表**（小标签样式）
- **发布日期**

#### 2.4 交互优化

- 键盘上下键导航结果列表
- 当前选中项高亮，回车打开
- 保留已有的 `Cmd+K` / `Ctrl+K` 快捷键和 `ESC` 关闭

涉及文件：
- `src/components/blog/Search.tsx`（主要重写）
- `src/app/api/search/route.ts`（返回 content + tags 字段）

### 三、页面装饰性背景

在前台页面添加 CSS 装饰元素，增加视觉层次感：

- **渐变光斑**：页面顶部和底部放置 2-3 个大尺寸模糊圆形（`blur-3xl`），使用低饱和度暖色调，与现有 surface 色协调
- **位置**：fixed 定位在视口中，不随页面滚动（或轻微视差效果），z-index 置于内容之下
- **暗色模式适配**：暗色模式下光斑换为低亮度冷色调
- **实现方式**：在根布局 `layout.tsx` 中添加装饰层，仅在非 admin 路径下渲染
- **性能**：纯 CSS 实现，使用 `will-change: transform` 优化合成层

涉及文件：
- `src/app/layout.tsx`（添加装饰层渲染逻辑）
- 新建 `src/components/blog/BackgroundDecor.tsx`（装饰元素组件）
