# 博客前台 + 管理后台 UI 优化

> 状态：进行中

## 背景

博客上线后，前台文章页和管理后台存在多处体验问题：标题重复、内容区过窄、代码块/标题对比度不足、缺少目录导航、管理后台颜色太浅。

## 方案

### 文章详情页

- 剥离 markdown 首行 `# 标题`，避免与页面 `<h1>` 重复
- 文章居中 `max-w-3xl`，左侧 absolute 定位 TOC 目录（xl 屏幕显示）
- TOC 从 DOM 读取 heading ID（`useSyncExternalStore`），保证与 `rehype-slug` 一致
- 标题 `font-light` → `font-semibold`，加 `text-ink` 颜色
- 代码块 Shiki 主题换为 `github-light-high-contrast`，保持浅色系但增加对比度
- 锚点跳转加 `scroll-margin-top` 避免被导航栏遮挡
- 回到顶部按钮放在 TOC 下方

### 全局色值

- 边框色持续加深（原 `#e8e5e0` → 目标清晰可辨）
- muted 文字色加深

### 管理后台

- 表单 label、输入框边框、辅助文字颜色加深
- 与博客前台统一风格
